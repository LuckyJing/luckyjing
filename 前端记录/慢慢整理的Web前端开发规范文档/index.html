慢慢整理的Web前端开发规范文档

发表于 2015年12月26日   |   分类于 前端记录

###通用规范

  • TAB键用两个空格代替（WINDOWS下TAB键占四个空格，LINUX下TAB键占八个空格）。
  • CSS样式属性或者JAVASCRIPT代码后加“;”方便压缩工具“断句”。
  • 文件内容编码均统一为UTF-8。
  • CSS、JAVASCRIPT中的非注释类中文字符须转换成unicode编码使用，以避免编码错误时乱码显示。

###文件规范

  • 多个单词用驼峰命名法
  • 避免拼音命名

###文档类型声明

  • <!DOCTYPE html>统一声明为html5类型
  • <meta charset="utf-8">编码统一为UTF-8
  • <meta http-equiv="X-UA-Compatible" content="IE=Edge">通知IE使用最新的内核渲染

###css和js文件的引用

###html属性的排列

  • class
  • id、name
  • data-*
  • src、for、type、href
  • title、alt
  • role
    • CLASS –> nHeadTitle –> CLASS遵循小驼峰命名法（little camel-case）
    • ID –> n_headtitle –> ID遵循名称+
    • NAME –> N_HeadTitle –> NAME属性命名遵循首个字母大写+

###表单元素input添加label

HTML中对于属性的定义全部使用双引号

###CSS书写顺序。

  • 兼容多个浏览器时，将标准属性写在底部。

  • 多选择器规则之间换行，即当样式针对多个选择器时每个选择器占一行。

###JavaScript规范

  • 常量名MY_NAME
  • 对象的属性或者方法名
    • 类名 Current
    • 函数名 defaultFun()
    • 变量名 currentValue
    • 私有变量名 _private

  • 下面类型的对象不建议用new构造

  • 从number到string的转换

    Math.round(x)//对x进行四舍五入,12.2返回12
    Math.ceil(x)//对浮点数进行向上取整，12.2返回13
